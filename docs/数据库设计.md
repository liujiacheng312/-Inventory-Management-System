# 库存管理系统数据库设计

## 1. 数据库概述

| 项目 | 内容 |
| --- | --- |
| 数据库类型 | MySQL |
| 推荐版本 | MySQL 8.0+ |
| 数据库名 | `inventory` |
| 字符集 | `utf8mb4` |
| 排序规则 | `utf8mb4_unicode_ci` |
| 初始化脚本 | `database/init.sql` |

本系统主要管理库存物料、用户、领取/归还/入库/导出记录，以及用户当前未归还数量。

当前代码实际使用的数据表：

| 表名 | 说明 | 当前初始化脚本状态 |
| --- | --- | --- |
| `users` | 用户表 | 已包含 |
| `materials` | 物料库存表 | 已包含 |
| `records` | 操作记录表 | 已包含 |
| `user_borrowed` | 用户未归还物料表 | 已包含 |

## 2. ER 关系

```text
users 1 ──── N records
users 1 ──── N user_borrowed

materials 1 ──── N records
materials 1 ──── N user_borrowed

users + materials 在 user_borrowed 中形成唯一借用状态：
UNIQUE(user_id, material_id)
```

说明：

- 一个用户可以产生多条操作记录。
- 一个物料可以被多条记录引用。
- 一个用户对同一个物料只保留一条未归还数量记录。
- Excel 导出记录的 `records.material_id` 可以为 `NULL`。

## 3. 表结构设计

### 3.1 users

用途：保存登录用户、密码哈希和角色。

建表 SQL：

```sql
CREATE TABLE IF NOT EXISTS users (
  id INT AUTO_INCREMENT PRIMARY KEY,
  username VARCHAR(50) NOT NULL UNIQUE,
  password VARCHAR(255) NOT NULL,
  role VARCHAR(20) NOT NULL DEFAULT 'user',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

字段说明：

| 字段 | 类型 | 允许为空 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| `id` | INT | 否 | 自增 | 主键 |
| `username` | VARCHAR(50) | 否 | 无 | 用户名，唯一 |
| `password` | VARCHAR(255) | 否 | 无 | bcrypt 加密后的密码 |
| `role` | VARCHAR(20) | 否 | `user` | 用户角色 |
| `created_at` | DATETIME | 是 | 当前时间 | 创建时间 |
| `updated_at` | DATETIME | 是 | 当前时间 | 更新时间 |

索引和约束：

| 名称 | 字段 | 类型 | 说明 |
| --- | --- | --- | --- |
| PRIMARY | `id` | 主键 | 用户唯一标识 |
| username | `username` | 唯一索引 | 防止用户名重复 |

角色枚举：

| 值 | 说明 |
| --- | --- |
| `super_admin` | 超级管理员 |
| `admin` | 管理员 |
| `user` | 普通用户 |

默认测试数据：

```sql
INSERT INTO users (username, password, role)
VALUES ('admin', '$2a$10$UQfYbuMUb/BL6hO0zi77tO7F3D2wNdwKdFTC9TZonzyXKHV6FGrai', 'super_admin');

INSERT INTO users (username, password, role)
VALUES ('user1', '$2a$10$UQfYbuMUb/BL6hO0zi77tO7F3D2wNdwKdFTC9TZonzyXKHV6FGrai', 'user');
```

默认密码：`admin123`。

注意：

- 密码必须保存 bcrypt 哈希，不能保存明文。
- 生产环境应修改默认账号密码。
- 当前后端启动时会执行 `bootstrapDatabase()`，确保至少存在一个 `super_admin`。

### 3.2 materials

用途：保存库存物料的基础信息和当前库存数量。

建表 SQL：

```sql
CREATE TABLE IF NOT EXISTS materials (
  id INT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(50) NOT NULL COMMENT '物料名称',
  brand VARCHAR(50) NOT NULL COMMENT '品牌',
  model VARCHAR(50) NOT NULL COMMENT '型号',
  spec VARCHAR(50) NOT NULL COMMENT '规格',
  quantity INT NOT NULL DEFAULT 0 COMMENT '数量',
  remark VARCHAR(255) COMMENT '备注',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY unique_item (name, brand, model, spec)
);
```

字段说明：

| 字段 | 类型 | 允许为空 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| `id` | INT | 否 | 自增 | 主键 |
| `name` | VARCHAR(50) | 否 | 无 | 物料名称 |
| `brand` | VARCHAR(50) | 否 | 无 | 品牌 |
| `model` | VARCHAR(50) | 否 | 无 | 型号 |
| `spec` | VARCHAR(50) | 否 | 无 | 规格 |
| `quantity` | INT | 否 | 0 | 当前库存数量 |
| `remark` | VARCHAR(255) | 是 | NULL | 备注 |
| `created_at` | DATETIME | 是 | 当前时间 | 创建时间 |
| `updated_at` | DATETIME | 是 | 当前时间 | 更新时间 |

索引和约束：

| 名称 | 字段 | 类型 | 说明 |
| --- | --- | --- | --- |
| PRIMARY | `id` | 主键 | 物料唯一标识 |
| unique_item | `name, brand, model, spec` | 唯一索引 | 判定同一物料 |

业务规则：

- 同一物料由 `name + brand + model + spec` 唯一确定。
- 管理员新增库存时，如果物料已存在，系统会累加 `quantity`。
- Excel 导入时也按该唯一组合判断新增或更新。
- `quantity` 业务上不应小于 0，扣减前必须校验库存是否足够。

### 3.3 records

用途：保存系统操作记录，包括入库、领取、归还和导出。

建表 SQL：

```sql
CREATE TABLE IF NOT EXISTS records (
  id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT NOT NULL,
  material_id INT DEFAULT NULL,
  count INT NOT NULL,
  type VARCHAR(10) NOT NULL COMMENT 'in=入库或归还, out=领取, export=导出',
  status VARCHAR(10) NOT NULL DEFAULT 'pending' COMMENT 'pending=待审批, approved=已通过, rejected=已拒绝',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id),
  FOREIGN KEY (material_id) REFERENCES materials(id)
);
```

字段说明：

| 字段 | 类型 | 允许为空 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| `id` | INT | 否 | 自增 | 主键 |
| `user_id` | INT | 否 | 无 | 操作用户 ID |
| `material_id` | INT | 是 | NULL | 物料 ID；导出记录可为空 |
| `count` | INT | 否 | 无 | 操作数量 |
| `type` | VARCHAR(10) | 否 | 无 | 操作类型 |
| `status` | VARCHAR(10) | 否 | `pending` | 审批状态 |
| `created_at` | DATETIME | 是 | 当前时间 | 创建时间 |
| `updated_at` | DATETIME | 是 | 当前时间 | 更新时间 |

外键：

| 字段 | 引用 | 说明 |
| --- | --- | --- |
| `user_id` | `users(id)` | 记录归属用户 |
| `material_id` | `materials(id)` | 记录关联物料 |

`type` 枚举：

| 值 | 说明 | 库存影响 |
| --- | --- | --- |
| `in` | 入库或归还 | 入库或归还审批通过后增加库存 |
| `out` | 领取 | 审批通过后减少库存 |
| `export` | 导出 Excel | 不影响库存 |

`status` 枚举：

| 值 | 说明 |
| --- | --- |
| `pending` | 待审批 |
| `approved` | 已通过 |
| `rejected` | 已拒绝 |

业务规则：

- 管理员新增库存会写入 `in / approved` 记录。
- 管理员领取会写入 `out / approved` 记录，并立即扣库存。
- 普通用户领取会写入 `out / pending` 记录，审批通过后扣库存。
- 管理员归还会写入 `in / approved` 记录，并立即加库存。
- 普通用户归还会写入 `in / pending` 记录，审批通过后加库存。
- Excel 导出会写入 `export / approved` 记录，`material_id` 为 `NULL`。

当前查询注意事项：

- 记录查询接口当前使用 `JOIN materials`，因此 `material_id IS NULL` 的导出记录可能不会出现在记录列表中。
- 如需展示导出记录，建议将查询改为 `LEFT JOIN materials`。

### 3.4 user_borrowed

用途：保存用户当前仍未归还的物料数量。

当前代码依赖该表：

- `GET /material` 查询当前用户 `borrowed_count`。
- `POST /material/take` 管理员直接领取或审批通过后会累加借用数量。
- `POST /material/return` 会校验并扣减借用数量。
- `GET /material/borrowed` 查询未归还物料。
- `POST /record/approve/:id` 审批领取/归还时会更新该表。

建议建表 SQL：

```sql
CREATE TABLE IF NOT EXISTS user_borrowed (
  id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT NOT NULL,
  material_id INT NOT NULL,
  count INT NOT NULL DEFAULT 0,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY unique_user_material (user_id, material_id),
  FOREIGN KEY (user_id) REFERENCES users(id),
  FOREIGN KEY (material_id) REFERENCES materials(id)
);
```

字段说明：

| 字段 | 类型 | 允许为空 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| `id` | INT | 否 | 自增 | 主键 |
| `user_id` | INT | 否 | 无 | 用户 ID |
| `material_id` | INT | 否 | 无 | 物料 ID |
| `count` | INT | 否 | 0 | 当前未归还数量 |
| `created_at` | DATETIME | 是 | 当前时间 | 创建时间 |
| `updated_at` | DATETIME | 是 | 当前时间 | 更新时间 |

索引和约束：

| 名称 | 字段 | 类型 | 说明 |
| --- | --- | --- | --- |
| PRIMARY | `id` | 主键 | 借用状态记录 ID |
| unique_user_material | `user_id, material_id` | 唯一索引 | 每个用户每个物料只保留一条状态 |

业务规则：

- 领取审批通过后：`count = count + 领取数量`。
- 归还审批通过后：`count = count - 实际归还数量`。
- 查询未归还物料时只返回 `count > 0`。
- 业务上 `count` 不应小于 0。

## 4. 完整推荐初始化 SQL

下面是与当前代码匹配的初始化结构。当前 `database/init.sql` 已包含 `user_borrowed` 表，新环境可直接执行初始化。

```sql
CREATE DATABASE IF NOT EXISTS inventory CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE inventory;

CREATE TABLE IF NOT EXISTS users (
  id INT AUTO_INCREMENT PRIMARY KEY,
  username VARCHAR(50) NOT NULL UNIQUE,
  password VARCHAR(255) NOT NULL,
  role VARCHAR(20) NOT NULL DEFAULT 'user',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS materials (
  id INT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(50) NOT NULL COMMENT '物料名称',
  brand VARCHAR(50) NOT NULL COMMENT '品牌',
  model VARCHAR(50) NOT NULL COMMENT '型号',
  spec VARCHAR(50) NOT NULL COMMENT '规格',
  quantity INT NOT NULL DEFAULT 0 COMMENT '数量',
  remark VARCHAR(255) COMMENT '备注',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY unique_item (name, brand, model, spec)
);

CREATE TABLE IF NOT EXISTS records (
  id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT NOT NULL,
  material_id INT DEFAULT NULL,
  count INT NOT NULL,
  type VARCHAR(10) NOT NULL COMMENT 'in=入库或归还, out=领取, export=导出',
  status VARCHAR(10) NOT NULL DEFAULT 'pending' COMMENT 'pending=待审批, approved=已通过, rejected=已拒绝',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id),
  FOREIGN KEY (material_id) REFERENCES materials(id)
);

CREATE TABLE IF NOT EXISTS user_borrowed (
  id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT NOT NULL,
  material_id INT NOT NULL,
  count INT NOT NULL DEFAULT 0,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY unique_user_material (user_id, material_id),
  FOREIGN KEY (user_id) REFERENCES users(id),
  FOREIGN KEY (material_id) REFERENCES materials(id)
);
```

## 5. 数据流说明

### 5.1 入库

触发接口：

```text
POST /material/add
POST /excel/import
```

数据变化：

1. 如果物料不存在，插入 `materials`。
2. 如果物料已存在，累加 `materials.quantity`。
3. 写入 `records`，`type = 'in'`，`status = 'approved'`。

### 5.2 普通用户领取

触发接口：

```text
POST /material/take
```

提交申请时：

1. 不扣减库存。
2. 不更新 `user_borrowed`。
3. 写入 `records`，`type = 'out'`，`status = 'pending'`。

审批通过时：

1. 检查 `materials.quantity >= records.count`。
2. 扣减 `materials.quantity`。
3. 累加 `user_borrowed.count`。
4. 更新 `records.status = 'approved'`。

审批拒绝时：

1. 不修改库存。
2. 不修改借用数量。
3. 更新 `records.status = 'rejected'`。

### 5.3 管理员领取

触发接口：

```text
POST /material/take
```

数据变化：

1. 检查库存是否足够。
2. 扣减 `materials.quantity`。
3. 累加 `user_borrowed.count`。
4. 写入 `records`，`type = 'out'`，`status = 'approved'`。

### 5.4 普通用户归还

触发接口：

```text
POST /material/return
```

提交申请时：

1. 查询 `user_borrowed`，确认存在可归还数量。
2. 写入 `records`，`type = 'in'`，`status = 'pending'`。
3. 不立即增加库存。

审批通过时：

1. 计算实际归还数量：`min(records.count, user_borrowed.count)`。
2. 增加 `materials.quantity`。
3. 扣减 `user_borrowed.count`。
4. 更新 `records.status = 'approved'`。

### 5.5 管理员归还

触发接口：

```text
POST /material/return
```

数据变化：

1. 查询当前用户可归还数量。
2. 计算实际归还数量。
3. 增加 `materials.quantity`。
4. 扣减 `user_borrowed.count`。
5. 写入 `records`，`type = 'in'`，`status = 'approved'`。

### 5.6 Excel 导出

触发接口：

```text
GET /excel/export
```

数据变化：

1. 不修改库存。
2. 写入 `records`，`type = 'export'`，`status = 'approved'`。
3. `material_id = NULL`。
4. `count = 导出的库存行数`。

## 6. 索引设计建议

当前表结构已经包含主键和核心唯一索引。随着数据增长，可增加以下索引：

```sql
CREATE INDEX idx_records_user_id ON records(user_id);
CREATE INDEX idx_records_material_id ON records(material_id);
CREATE INDEX idx_records_status_created_at ON records(status, created_at);
CREATE INDEX idx_records_created_at ON records(created_at);
CREATE INDEX idx_materials_name ON materials(name);
```

适用场景：

| 索引 | 适用查询 |
| --- | --- |
| `idx_records_user_id` | 普通用户查询自己的记录 |
| `idx_records_material_id` | 查询某个物料的历史记录 |
| `idx_records_status_created_at` | 管理员查询待审批记录 |
| `idx_records_created_at` | 按时间倒序查询记录 |
| `idx_materials_name` | 物料名称搜索或排序 |

## 7. 数据一致性建议

当前库存、记录、借用状态在多个 SQL 语句中更新，建议后续对以下流程增加数据库事务：

- 管理员领取：扣库存、写借用、写记录。
- 管理员归还：加库存、扣借用、写记录。
- 审批通过领取：扣库存、写借用、改记录状态。
- 审批通过归还：加库存、扣借用、改记录状态。
- Excel 批量导入：多行物料写入和记录写入。

事务示例方向：

```sql
START TRANSACTION;
-- 更新库存
-- 更新 user_borrowed
-- 写入或更新 records
COMMIT;
```

失败时：

```sql
ROLLBACK;
```

## 10. 维护注意事项

1. 修改表字段后，要同步更新后端 SQL、前端模型和接口文档。
2. 物料唯一键不可随意删除，否则 Excel 导入和新增库存无法正确累加。
3. 删除物料前要确认是否存在历史记录和未归还数量。
4. 不建议物理删除用户；如有停用需求，后续可增加 `status` 字段。
5. 生产环境不要保留默认密码。
6. 数据库连接密码应迁移到环境变量或配置文件，不建议硬编码。
7. `records` 是审计数据，建议只追加和更新状态，不随意删除。
