# 库存管理系统接口文档

## 1. 基本信息

| 项目 | 内容 |
| --- | --- |
| 服务名称 | 库存管理系统后端 API |
| 后端框架 | Node.js + Express |
| 默认地址 | `http://localhost:3000` |
| 数据格式 | JSON |
| 文件上传 | multipart/form-data |
| 鉴权方式 | JWT Bearer Token |

本文档按当前代码中的路由实现编写，对应目录：

```text
server/routes/
├─ userRoutes.js
├─ materialRoutes.js
├─ recordRoutes.js
└─ excelRoutes.js
```

## 2. 通用约定

### 2.1 请求头

除注册、登录和根路径外，业务接口通常需要登录后访问。

```http
Authorization: Bearer <token>
Content-Type: application/json
```

Excel 导入接口使用 `multipart/form-data`，不需要手动设置 JSON Content-Type。

Excel 导出接口支持两种 token 传递方式：

```http
Authorization: Bearer <token>
```

或：

```text
GET /excel/export?token=<token>
```

### 2.2 通用成功响应

```json
{
  "success": true,
  "data": {},
  "message": "操作成功"
}
```

不同接口的 `data` 结构不同；部分接口只返回 `success` 和 `message`。

### 2.3 通用失败响应

```json
{
  "success": false,
  "message": "错误原因"
}
```

### 2.4 常见 HTTP 状态码

| 状态码 | 含义 | 常见场景 |
| --- | --- | --- |
| 200 | 请求成功 | 查询、新增、审批成功 |
| 201 | 创建成功 | 注册成功 |
| 400 | 请求参数错误 | 缺少必填字段、数量非法、文件类型错误 |
| 401 | 未认证 | 缺少 token、token 无效或过期、账号密码错误 |
| 403 | 无权限 | 普通用户访问管理员接口 |
| 404 | 资源不存在 | 物料不存在、审批记录不存在 |
| 409 | 数据冲突 | 注册用户名已存在 |
| 500 | 服务器内部错误 | 数据库异常、文件处理异常 |

### 2.5 角色权限

| 角色值 | 说明 | 权限摘要 |
| --- | --- | --- |
| `super_admin` | 超级管理员 | 管理员所有权限；可查看用户列表、修改用户角色 |
| `admin` | 管理员 | 库存管理、审批、Excel 导入导出 |
| `user` | 普通用户 | 查看库存、提交领取/归还申请、查看自己的记录 |

后端权限中间件：

| 中间件 | 说明 |
| --- | --- |
| `authMiddleware` | 校验 JWT，成功后写入 `req.user` |
| `adminMiddleware` | 允许 `admin` 和 `super_admin` |
| `superAdminMiddleware` | 只允许 `super_admin` |

## 4. 用户接口

### 4.1 注册

```http
POST /user/register
```

权限：公开

请求体：

```json
{
  "username": "user2",
  "password": "admin123"
}
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| username | string | 是 | 用户名，去除首尾空格后长度 3 到 20 |
| password | string | 是 | 密码，长度至少 6 |

成功响应：

```json
{
  "success": true,
  "message": "注册成功",
  "data": {
    "token": "<jwt>",
    "user": {
      "id": 3,
      "username": "user2",
      "role": "user"
    }
  }
}
```

失败场景：

| 状态码 | 场景 |
| --- | --- |
| 400 | 用户名或密码为空、用户名长度不符合要求、密码太短 |
| 409 | 用户名已存在 |
| 500 | 数据库或服务异常 |

### 4.2 登录

```http
POST /user/login
```

权限：公开

请求体：

```json
{
  "username": "admin",
  "password": "admin123"
}
```

成功响应：

```json
{
  "success": true,
  "data": {
    "token": "<jwt>",
    "user": {
      "id": 1,
      "username": "admin",
      "role": "super_admin"
    }
  }
}
```

说明：

- JWT 有效期为 24 小时。
- token payload 包含 `id`、`username`、`role`。

失败场景：

| 状态码 | 场景 |
| --- | --- |
| 400 | 用户名或密码为空 |
| 401 | 用户名或密码错误 |
| 500 | 数据库或服务异常 |

### 4.3 获取用户列表

```http
GET /user/list
```

权限：超级管理员

请求头：

```http
Authorization: Bearer <token>
```

成功响应：

```json
{
  "success": true,
  "data": [
    {
      "id": 1,
      "username": "admin",
      "role": "super_admin",
      "created_at": "2026-04-14T06:00:00.000Z"
    },
    {
      "id": 2,
      "username": "user1",
      "role": "user",
      "created_at": "2026-04-14T06:00:00.000Z"
    }
  ]
}
```

排序规则：

1. 超级管理员优先。
2. 管理员其次。
3. 普通用户最后。
4. 同角色按创建时间和 ID 升序。

失败场景：

| 状态码 | 场景 |
| --- | --- |
| 401 | 未登录或 token 无效 |
| 403 | 当前用户不是超级管理员 |
| 500 | 查询失败 |

### 4.4 修改用户角色

```http
PATCH /user/:id/role
```

权限：超级管理员

路径参数：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| id | number | 被修改用户 ID |

请求体：

```json
{
  "role": "admin"
}
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| role | string | 是 | 只能是 `admin` 或 `user` |

成功响应：

```json
{
  "success": true,
  "message": "已提升为管理员",
  "data": {
    "id": 2,
    "username": "user1",
    "role": "admin"
  }
}
```

规则：

- 不能修改自己的角色。
- 不能修改超级管理员角色。
- 只能在 `admin` 和 `user` 之间调整。

失败场景：

| 状态码 | 场景 |
| --- | --- |
| 400 | 用户 ID 非法、角色非法、修改自己 |
| 401 | 未登录或 token 无效 |
| 403 | 不是超级管理员，或目标用户是超级管理员 |
| 404 | 目标用户不存在 |
| 500 | 更新失败 |

## 5. 库存接口

### 5.1 获取库存列表

```http
GET /material
```

权限：登录用户

成功响应：

```json
{
  "success": true,
  "data": [
    {
      "id": 1,
      "name": "螺丝刀",
      "brand": "世达",
      "model": "A1",
      "spec": "6mm",
      "quantity": 100,
      "remark": "常用工具",
      "borrowed_count": 2
    }
  ]
}
```

字段说明：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | number | 物料 ID |
| name | string | 物料名称 |
| brand | string | 品牌 |
| model | string | 型号 |
| spec | string | 规格 |
| quantity | number | 当前库存数量 |
| remark | string | 备注 |
| borrowed_count | number | 当前登录用户未归还数量 |

排序规则：按物料名称首字符、物料名称、品牌、型号、规格排序。

注意：该接口依赖 `user_borrowed` 表，当前 `database/init.sql` 已包含该表。

### 5.2 新增库存

```http
POST /material/add
```

权限：管理员

请求体：

```json
{
  "name": "螺丝刀",
  "brand": "世达",
  "model": "A1",
  "spec": "6mm",
  "quantity": 100,
  "remark": "常用工具"
}
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| name | string | 是 | 物料名称 |
| brand | string | 是 | 品牌 |
| model | string | 是 | 型号 |
| spec | string | 是 | 规格 |
| quantity | number | 是 | 入库数量 |
| remark | string | 否 | 备注 |

成功响应：

```json
{
  "success": true,
  "message": "添加成功"
}
```

如果 `name + brand + model + spec` 已存在，则累加库存并返回：

```json
{
  "success": true,
  "message": "库存已更新（累加）"
}
```

业务规则：

- 新增或累加库存后，会写入一条 `records` 记录。
- 记录类型为 `in`。
- 记录状态为 `approved`。

失败场景：

| 状态码 | 场景 |
| --- | --- |
| 400 | 缺少必填字段 |
| 401 | 未登录或 token 无效 |
| 403 | 不是管理员 |
| 500 | 数据库异常 |


权限：管理员

路径参数：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| id | number | 物料 ID |

成功响应：

```json
{
```


注意：

### 5.5 领取物料

```http
POST /material/take
```

权限：登录用户

请求体：

```json
{
  "material_id": 1,
  "count": 2
}
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| material_id | number | 是 | 物料 ID |
| count | number | 是 | 领取数量，必须大于 0 |

管理员成功响应：

```json
{
  "success": true,
  "message": "领取成功，库存已自动减少"
}
```

普通用户成功响应：

```json
{
  "success": true,
  "message": "申请已提交，等待管理员审批"
}
```

业务规则：

- 管理员领取会立即扣减库存，写入 `user_borrowed`，记录状态为 `approved`。
- 普通用户领取只写入待审批记录，记录状态为 `pending`。
- 管理员领取时会校验库存是否足够。
- 普通用户提交时当前代码不立即扣库存，最终由审批接口校验库存。

失败场景：

| 状态码 | 场景 |
| --- | --- |
| 400 | 缺少物料 ID、数量非法、管理员领取时库存不足 |
| 404 | 物料不存在 |
| 500 | 数据库异常 |

### 5.6 归还物料

```http
POST /material/return
```

权限：登录用户

请求体：

```json
{
  "material_id": 1,
  "count": 1
}
```

管理员成功响应：

```json
{
  "success": true,
  "message": "归还成功，库存已自动增加"
}
```

普通用户成功响应：

```json
{
  "success": true,
  "message": "归还申请已提交，等待管理员审批"
}
```

业务规则：

- 系统会先查询当前用户在 `user_borrowed` 中的未归还数量。
- 实际归还数量为 `min(请求数量, 未归还数量)`。
- 管理员归还会立即增加库存并扣减借用数量。
- 普通用户归还会写入待审批记录，审批通过后再更新库存和借用数量。

失败场景：

| 状态码 | 场景 |
| --- | --- |
| 400 | 参数非法、没有可归还物品 |
| 404 | 物料不存在 |
| 500 | 数据库异常 |

## 6. 记录接口

### 6.1 获取操作记录

```http
GET /record
```

权限：登录用户

管理员响应示例：

```json
{
  "success": true,
  "data": [
    {
      "id": 1,
      "user_id": 2,
      "material_id": 1,
      "count": 2,
      "type": "out",
      "status": "pending",
      "created_at": "2026-04-14T06:00:00.000Z",
      "updated_at": "2026-04-14T06:00:00.000Z",
      "username": "user1",
      "name": "螺丝刀",
      "brand": "世达",
      "spec": "6mm"
    }
  ]
}
```

普通用户响应结构基本相同，但只返回自己的记录，且不包含 `username`。

业务规则：

- 管理员和超级管理员查看全部记录。
- 普通用户只查看自己的记录。
- 当前查询使用 `JOIN materials`，`material_id` 为空的导出记录可能不会出现在记录列表中。

### 6.2 获取待审批记录

```http
GET /record/pending
```

权限：管理员

成功响应：

```json
{
  "success": true,
  "data": [
    {
      "id": 1,
      "user_id": 2,
      "material_id": 1,
      "count": 2,
      "type": "out",
      "status": "pending",
      "username": "user1",
      "name": "螺丝刀",
      "brand": "世达",
      "spec": "6mm",
      "created_at": "2026-04-14T06:00:00.000Z"
    }
  ]
}
```

说明：只返回 `status = 'pending'` 的记录。

### 6.3 审批通过

```http
POST /record/approve/:id
```

权限：管理员

路径参数：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| id | number | 待审批记录 ID |

成功响应：

```json
{
  "success": true,
  "message": "审批成功"
}
```

业务规则：

- 仅能审批 `status = 'pending'` 的记录。
- `type = 'out'`：校验库存，库存足够则扣减库存并累加 `user_borrowed`。
- `type = 'in'`：按用户未归还数量计算实际归还数量，增加库存并扣减 `user_borrowed`。
- 审批成功后将记录状态更新为 `approved`。
- 领取审批时如果库存不足，记录会被自动改为 `rejected`，并返回失败。

失败场景：

| 状态码 | 场景 |
| --- | --- |
| 400 | 领取审批时库存不足 |
| 401 | 未登录或 token 无效 |
| 403 | 不是管理员 |
| 404 | 记录不存在或已处理 |
| 500 | 审批失败 |

### 6.4 审批拒绝

```http
POST /record/reject/:id
```

权限：管理员

路径参数：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| id | number | 待审批记录 ID |

成功响应：

```json
{
  "success": true,
  "message": "已拒绝申请"
}
```

说明：当前实现会直接把指定记录状态更新为 `rejected`，不校验原状态是否为 `pending`。

## 7. Excel 接口

### 7.1 导入 Excel

```http
POST /excel/import
```

权限：管理员

请求类型：`multipart/form-data`

表单字段：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| file | File | 是 | `.xlsx` 文件 |

cURL 示例：

```bash
curl -X POST "http://localhost:3000/excel/import" \
  -H "Authorization: Bearer <token>" \
  -F "file=@inventory.xlsx"
```

成功响应：

```json
{
  "success": true,
  "data": {
    "successCount": 3,
    "updateCount": 2,
    "total": 5
  },
  "message": "导入完成：新增 3 条，更新 2 条。"
}
```

支持表头：

| 含义 | 支持表头 |
| --- | --- |
| 物料名称 | `物料名称`、`物品`、`name`、`Name` |
| 品牌 | `品牌`、`brand`、`Brand` |
| 型号 | `型号`、`model`、`Model` |
| 规格 | `规格`、`spec`、`Spec`、`specification` |
| 数量 | `数量`、`库存数量`、`quantity`、`Quantity` |
| 备注 | `备注`、`remark`、`Remark` |

业务规则：

- 只支持 `.xlsx`。
- 读取第一个工作表。
- 缺少物料名称、品牌或规格的行会被跳过。
- 数量无法解析时按 0 处理。
- 已存在的物料按 `name + brand + model + spec` 判定，并累加库存。
- 每条新增或更新都会写入一条 `in / approved` 记录。
- 导入结束后删除上传临时文件。

失败场景：

| 状态码 | 场景 |
| --- | --- |
| 400 | 文件缺失或文件类型不支持 |
| 401 | 未登录或 token 无效 |
| 403 | 不是管理员 |
| 500 | Excel 解析或数据库写入失败 |

### 7.2 导出 Excel

```http
GET /excel/export
```

权限：管理员

请求示例：

```text
GET http://localhost:3000/excel/export?token=<token>
```

或：

```bash
curl -L "http://localhost:3000/excel/export" \
  -H "Authorization: Bearer <token>" \
  -o 库存数据.xlsx
```

成功响应：返回 Excel 文件下载。

导出字段：

| Excel 列 | 数据来源 |
| --- | --- |
| 序号 | 导出时生成 |
| 物料名称 | `materials.name` |
| 品牌 | `materials.brand` |
| 型号 | `materials.model` |
| 规格 | `materials.spec` |
| 数量 | `materials.quantity` |
| 备注 | `materials.remark` |

业务规则：

- 查询全部库存并按名称、品牌、型号、规格排序。
- 生成临时 `.xlsx` 文件。
- 写入一条 `export / approved` 记录，`material_id` 为 `NULL`，`count` 为导出物料数量。
- 下载后临时文件会在约 60 秒后删除。

失败场景：

| 状态码 | 场景 |
| --- | --- |
| 401 | 缺少 token、token 无效或过期 |
| 403 | 不是管理员 |
| 500 | 查询、生成或下载失败 |

## 9. 联调检查清单

1. 后端服务已启动，并能访问 `GET /`。
2. MySQL 已创建 `inventory` 数据库。
3. `users` 表存在默认账号。
4. 需要领取/归还功能时，确认数据库中存在 `user_borrowed` 表；当前初始化脚本已包含该表。
5. 前端 `ApiConfig.baseUrl` 与后端地址一致。
6. 登录成功后，前端后续请求都携带 `Authorization`。
7. 管理员功能使用 `admin` 或 `super_admin` 账号测试。
8. Excel 导入使用 `.xlsx` 文件，表单字段名必须是 `file`。
