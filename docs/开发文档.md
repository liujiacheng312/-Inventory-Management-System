# 库存管理系统开发文档

## 1. 文档目标

本文档用于帮助开发者快速理解项目结构、关键页面职责、接口调用方式和近期代码调整方向，方便继续迭代。

## 2. 技术栈

| 模块 | 技术 |
| --- | --- |
| 前端 | Flutter 3.x / Dart |
| 前端依赖 | `http`、`shared_preferences`、`file_picker`、`desktop_drop`、`cross_file`、`url_launcher` |
| 后端 | Node.js / Express |
| 数据库 | MySQL 8.0+ |
| 鉴权 | JWT Bearer Token |
| 文件处理 | `multer` + `xlsx` |

## 4. 本地开发启动

### 4.1 启动后端

```bash
cd server
npm install
npm run dev
```

默认后端地址：

```text
http://localhost:3000
```

后端入口文件是 `server/app.js`，当前默认监听 `0.0.0.0`，所以也支持局域网访问。

### 4.2 启动前端

```bash
cd client/my_app
flutter pub get
flutter run -d windows
```

如果要连接局域网后端，使用：

```bash
flutter run -d windows --dart-define=API_BASE_URL=http://192.168.1.100:3000
```

### 4.3 Android 调试与打包

```bash
cd client/my_app
flutter run -d android --dart-define=API_BASE_URL=http://192.168.1.100:3000
flutter build apk --release --dart-define=API_BASE_URL=http://192.168.1.100:3000
```

当前项目在 Android `release` 构建时建议使用 JDK 17。


## 7. 关键接口约定

### 7.1 通用响应结构

成功时通常返回：

```json
{
  "success": true,
  "data": {},
  "message": "操作成功"
}
```

失败时通常返回：

```json
{
  "success": false,
  "message": "错误原因"
}
```

需要登录的接口统一使用：

```http
Authorization: Bearer <token>
```

### 7.2 关键接口

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | `/user/register` | 注册普通用户 |
| POST | `/user/login` | 登录并返回 `token + user` |
| GET | `/material` | 获取库存列表 |
| POST | `/material/take` | 领用物料 |
| POST | `/material/return` | 归还物料 |
| GET | `/record` | 获取操作记录 |
| GET | `/record/pending` | 获取待审批记录 |
| POST | `/record/approve/:id` | 批准待审批记录 |
| POST | `/record/reject/:id` | 拒绝待审批记录 |
| POST | `/excel/import` | 导入 `.xlsx` |
| GET | `/excel/export` | 导出库存 Excel |

## 8. 数据流说明

### 8.1 登录

1. 登录页调用 `ApiService.login`
2. 后端校验用户名与 bcrypt 密码
3. 成功后返回 `token` 和 `user`
4. 前端通过 `User.fromJson` 构造登录用户
5. 后续请求统一带上 Bearer Token

### 8.2 领用与归还

- 管理员提交后直接生效
- 普通用户提交后进入 `pending`
- 管理员在待审批页统一处理
- 审批通过后同步更新库存和借用数量

### 8.3 Excel 导入导出

- 导入使用 multipart/form-data，字段名为 `file`
- 导入文件格式为 `.xlsx`
- 导出接口会返回 Excel 文件下载链接或文件流

## 11. 开发注意事项

### 11.1 后端

- 新增需要登录的接口时先挂 `authMiddleware`
- 管理员接口使用管理员中间件，不要只做前端限制
- SQL 统一使用参数化查询
- 改动库存逻辑时同时检查 `materials`、`records`、`user_borrowed`

### 11.2 前端

- API 调用集中写在 `ApiService`
- 新增页面优先复用 `AppTheme`
- 角色判断统一通过 `User` 模型上的 `isAdmin` / `isSuperAdmin`
- 手机端优先检查窄屏下的首屏高度，不要默认沿用桌面卡片尺寸

### 11.3 配置

- `server/config/db.js` 当前仍是硬编码数据库配置
- `JWT_SECRET` 当前仍未迁移到环境变量
- 后续若准备公开部署，建议先把敏感配置挪出仓库

## 12. 常见开发问题

### 12.1 登录不了

优先检查：

- 后端是否启动
- API 地址是否正确
- 默认账号是否使用 `admin / admin123`
- 客户端是否构建了最新包

### 12.2 其他设备访问不到后端

优先检查：

- 是否使用了服务器内网 IP，而不是 `localhost`
- Windows 防火墙是否放行 `3000`
- 手机和电脑是否在同一局域网

### 12.3 Android Release 构建失败

优先检查：

- 是否切换到 JDK 17
- 是否重新执行 `flutter pub get`
- 是否存在旧 Gradle 缓存冲突

## 13. 后续建议

- 把数据库配置和 JWT 密钥迁移到环境变量
- 为关键库存与审批流程补自动化测试
- 将 `routes` 中的业务逻辑逐步拆到 service/controller 层
- 正式分发 Android 版本前接入正式签名和 AAB 流程
