# 库存管理系统部署说明

## 1. 部署目标

本文档说明库存管理系统从本地开发到服务器部署的完整步骤，覆盖：

- MySQL 数据库初始化
- Node.js 后端部署
- Flutter 前端运行和构建
- 局域网访问配置
- 生产环境配置建议
- 常见问题排查

项目结构：

```text
bz/
├─ database/
│  └─ init.sql
├─ server/
│  ├─ app.js
│  ├─ package.json
│  └─ config/db.js
└─ client/
   └─ my_app/
      ├─ pubspec.yaml
      └─ lib/config/api.dart
```

## 3. 本地开发部署

### 3.1 初始化数据库

确认 MySQL 服务已经启动。

在项目根目录执行：

```bash
mysql -u root -p < database/init.sql
```

如果需要手动创建数据库：

```sql
CREATE DATABASE IF NOT EXISTS inventory CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

然后进入数据库执行初始化脚本。

当前 `database/init.sql` 已包含领取/归还流程依赖的 `user_borrowed` 表。若旧环境缺失该表，可执行以下补丁：

```sql
USE inventory;

CREATE TABLE IF NOT EXISTS user_borrowed (
  id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT NOT NULL,
  material_id INT NOT NULL,
  count INT NOT NULL DEFAULT 0,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY unique_user_material (user_id, material_id),
  FOREIGN KEY (user_id) REFERENCES users(id),
  FOREIGN KEY (material_id) REFERENCES materials(id)
);
```

### 3.2 配置数据库连接

编辑：

```text
server/config/db.js
```

本地示例：

```js
const dbConfig = {
  host: 'localhost',
  user: 'root',
  password: '你的数据库密码',
  database: 'inventory',
  waitForConnections: true,
  connectionLimit: 10,
  queueLimit: 0
};
```

配置说明：

| 字段 | 说明 |
| --- | --- |
| `host` | MySQL 主机地址 |
| `user` | MySQL 用户名 |
| `password` | MySQL 密码 |
| `database` | 数据库名，默认 `inventory` |
| `connectionLimit` | 连接池最大连接数 |

## 5. 局域网部署

适用场景：后端和数据库部署在一台局域网服务器，其他电脑运行 Flutter 客户端访问。

### 5.1 确认服务器 IP

Windows：

```bash
ipconfig
```

Linux：

```bash
ip addr
```

假设服务器 IP 为：

```text
192.168.1.100
```

### 5.2 修改后端监听

当前后端代码：

```js
app.listen(3000, () => {
  console.log('Server running on http://localhost:3000');
});
```

Node.js 默认会监听所有网卡或本机环境可访问地址。若局域网无法访问，可明确改为：

```js
app.listen(3000, '0.0.0.0', () => {
  console.log('Server running on http://0.0.0.0:3000');
});
```

### 5.3 配置防火墙

需要放行 TCP 3000 端口。

Windows 防火墙可创建入站规则：

- 类型：端口
- 协议：TCP
- 端口：3000
- 操作：允许连接

### 5.4 修改前端地址

```dart
class ApiConfig {
  static const String baseUrl = 'http://192.168.1.100:3000';
}
```

然后重新运行或重新构建前端。

### 5.5 测试访问

在客户端电脑浏览器访问：

```text
http://192.168.1.100:3000/
```

能看到 API 信息表示网络可达。

## 6. Linux 服务器部署

适用场景：后端部署在 Linux 服务器，前端可以使用 Web 或桌面客户端访问。

### 6.1 安装 Node.js

Ubuntu 示例：

```bash
sudo apt update
sudo apt install -y nodejs npm
node -v
npm -v
```

如果系统源版本过低，建议使用 NodeSource 安装 Node.js 18+。

### 6.2 安装 MySQL

```bash
sudo apt install -y mysql-server
sudo systemctl enable mysql
sudo systemctl start mysql
```

登录 MySQL：

```bash
sudo mysql
```

创建数据库用户示例：

```sql
CREATE DATABASE IF NOT EXISTS inventory CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER IF NOT EXISTS 'inventory_user'@'localhost' IDENTIFIED BY '强密码';
GRANT ALL PRIVILEGES ON inventory.* TO 'inventory_user'@'localhost';
FLUSH PRIVILEGES;
```

导入初始化脚本：

```bash
mysql -u inventory_user -p inventory < database/init.sql
```

当前脚本已包含 `user_borrowed`。如果部署的是旧版本数据库，按本文第 3.1 节的补丁确认该表存在。

### 6.3 上传代码

可通过 Git、SCP、压缩包等方式把项目放到服务器，例如：

```text
/opt/inventory-system
```

### 6.4 安装后端依赖

```bash
cd /opt/inventory-system/server
npm install --production
```

### 6.5 配置数据库连接

编辑：

```text
/opt/inventory-system/server/config/db.js
```

示例：

```js
const dbConfig = {
  host: 'localhost',
  user: 'inventory_user',
  password: '强密码',
  database: 'inventory',
  waitForConnections: true,
  connectionLimit: 10,
  queueLimit: 0
};
```

生产环境建议改造为环境变量，例如：

```text
DB_HOST=localhost
DB_USER=inventory_user
DB_PASSWORD=强密码
DB_NAME=inventory
JWT_SECRET=更长的随机密钥
PORT=3000
```

当前代码尚未读取 `.env`，如需启用环境变量，需要先改造 `server/config/db.js` 和 `server/middleware/auth.js`。

### 6.6 使用 PM2 运行后端

安装 PM2：

启动：

## 8. 后端反向代理

生产环境建议用 Nginx 反向代理 Node.js 服务，并配置 HTTPS。

### 8.1 HTTP 代理示例

```nginx
server {
    listen 80;
    server_name api.example.com;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

### 8.2 HTTPS 建议

可以使用 Certbot 申请 Let's Encrypt 证书：

```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d api.example.com
```

申请成功后，将前端 `baseUrl` 改为：

```dart
static const String baseUrl = 'https://api.example.com';
```

## 9. CORS 配置

当前后端使用：

```js
app.use(cors());
```

这表示允许所有来源访问，开发阶段方便联调。

生产环境建议限制来源：

```js
app.use(cors({
  origin: ['https://inventory.example.com'],
  credentials: true
}));
```

如果只使用桌面客户端或内网访问，可以按实际情况调整。

## 10. 文件目录权限

后端运行时会使用：

```text
server/uploads/
server/exports/
```

用途：

| 目录 | 用途 |
| --- | --- |
| `uploads` | Excel 导入临时文件 |
| `exports` | Excel 导出临时文件 |

部署时需要确保后端进程对这两个目录有读写权限。

Linux 示例：

```bash
mkdir -p /opt/inventory-system/server/uploads
mkdir -p /opt/inventory-system/server/exports
chown -R nodeuser:nodeuser /opt/inventory-system/server/uploads
chown -R nodeuser:nodeuser /opt/inventory-system/server/exports
```

当前代码会自动创建这两个目录，但如果父目录权限不足，仍会失败。

## 11. 安全配置建议

### 11.1 修改默认密码

部署后立刻修改默认账号密码，或删除默认测试账号。

### 11.2 修改 JWT 密钥

当前密钥在：

```text
server/middleware/auth.js
```

当前代码中为硬编码：

```js
const JWT_SECRET = 'inventory-system-secret-key-2024';
```

生产环境建议改为环境变量，并使用足够长的随机字符串。

### 11.3 数据库最小权限

不要使用 MySQL `root` 账号运行生产系统。建议创建专用用户：

```sql
CREATE USER 'inventory_user'@'localhost' IDENTIFIED BY '强密码';
GRANT SELECT, INSERT, UPDATE, DELETE, CREATE, ALTER, INDEX
ON inventory.* TO 'inventory_user'@'localhost';
FLUSH PRIVILEGES;
```

### 11.4 限制端口暴露

生产环境建议：

- MySQL 只监听内网或 localhost。
- Node.js 只监听本机，由 Nginx 对外代理。
- 只开放 80/443 或必要端口。

## 15. 常见问题排查

### 15.1 后端启动失败

检查：

```bash
node -v
npm install
npm start
```

常见原因：

- Node.js 版本过低。
- 依赖未安装。
- 3000 端口被占用。
- 数据库连接失败。
- 数据库初始化未完成。

查看端口占用：

Windows：

```powershell
netstat -ano | findstr :3000
```

Linux：

```bash
lsof -i :3000
```

### 15.2 数据库连接失败

检查：

- MySQL 服务是否启动。
- `server/config/db.js` 中账号、密码、数据库名是否正确。
- 数据库用户是否有权限。
- 服务器防火墙是否阻止 MySQL 连接。

测试连接：

```bash
mysql -u root -p inventory
```

### 15.3 登录失败

检查：

- `users` 表是否有默认账号。
- 密码是否为 `admin123`。
- 后端是否连接到了正确数据库。
- 密码字段是否为 bcrypt 哈希。

### 15.4 库存列表报错

重点检查：

```sql
SHOW TABLES;
```

确认存在：

```text
user_borrowed
```

如果不存在，执行本文第 3.1 节中的建表 SQL。

### 15.5 前端无法访问后端

检查：

- 后端是否启动。
- `ApiConfig.baseUrl` 是否正确。
- 桌面客户端和后端是否在同一台机器。
- 局域网访问时是否使用服务器 IP，而不是 `localhost`。
- 防火墙是否放行 3000 端口。

### 15.6 Excel 导入失败

检查：

- 文件是否为 `.xlsx`。
- 表单字段名是否为 `file`。
- 当前账号是否为管理员。
- `server/uploads` 是否有写权限。
- Excel 表头是否包含物料名称、品牌、规格和数量。

### 15.7 Excel 导出失败

检查：

- 当前账号是否为管理员。
- token 是否有效。
- `server/exports` 是否有写权限。
- 浏览器是否拦截下载。

## 17. 版本更新流程

推荐流程：

1. 备份数据库。
2. 停止或保留旧服务运行。
3. 拉取或上传新代码。
4. 检查数据库变更脚本。
5. 安装后端依赖。
6. 重新构建前端。
7. 重启后端。
8. 执行登录、库存列表、领取、归还、审批、Excel 导入导出冒烟测试。
9. 确认无误后清理旧版本文件。

命令示例：

```bash
mysqldump -u inventory_user -p inventory > /backup/inventory_before_update.sql
cd /opt/inventory-system
git pull
cd server
npm install --production
pm2 restart inventory-server
```

前端 Web 更新：

```bash
cd client/my_app
flutter pub get
flutter build web
sudo rsync -av --delete build/web/ /var/www/inventory-web/
sudo systemctl reload nginx
```
