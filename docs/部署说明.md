# 库存管理系统部署说明

## 1. 适用场景

本文档面向以下部署方式：

- Windows 单机运行
- 后端在局域网内网运行，前端从其他电脑或手机访问
- Flutter Windows 客户端调试
- Flutter Android APK 打包与内网测试安装

当前项目采用“Flutter 前端 + Node.js/Express 后端 + MySQL 数据库”的结构。后端默认监听 `0.0.0.0:3000`，适合放在内网服务器或办公室电脑中运行。

## 2. 环境要求

| 组件 | 推荐版本 | 说明 |
| --- | --- | --- |
| Node.js | 18+ | 运行后端服务 |
| npm | 随 Node.js 安装 | 安装后端依赖 |
| MySQL | 8.0+ | 保存业务数据 |
| Flutter | 3.x | 构建桌面端与 Android 客户端 |
| Dart SDK | 随 Flutter 安装 | Flutter 编译环境 |
| Android SDK | 最新稳定版 | Android 打包 |
| JDK | 17 | Android `release` 构建建议使用 |

## 4. 初始化数据库

在项目根目录执行：

```bash
mysql -u root -p < database/init.sql
```

如果数据库还没有创建，可先执行：

```sql
CREATE DATABASE IF NOT EXISTS inventory
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;
```

后端数据库配置文件位于：

```text
server/config/db.js
```

当前项目默认读取这里的硬编码配置。部署到新机器前，请先把 `host`、`user`、`password`、`database` 改成目标环境的实际值。

## 5. 启动后端

进入后端目录并安装依赖：

```bash
cd server
npm install
```

开发模式：

```bash
npm run dev
```

普通启动：

```bash
npm start
```

后端入口文件是 `server/app.js`，默认行为如下：

- 监听地址：`0.0.0.0`
- 监听端口：`3000`
- 支持通过环境变量覆盖 `HOST` 和 `PORT`
- 启动成功后会打印本机可用的局域网访问地址

如果需要手动指定：

```powershell
$env:HOST='0.0.0.0'
$env:PORT='3000'
npm start
```

验证服务是否启动成功：

```bash
curl http://127.0.0.1:3000/
```

如果返回 API 信息，说明后端正常。

## 6. 配置前端 API 地址

前端 API 配置位于：

```text
client/my_app/lib/config/api.dart
```

当前实现如下：

```dart
class ApiConfig {
  static const String baseUrl = String.fromEnvironment(
    'API_BASE_URL',
    defaultValue: 'http://localhost:3000',
  );
}
```

这意味着：

- 前后端在同一台机器上运行时，不传参数也可以，默认走 `http://localhost:3000`
- 前端运行在局域网其他电脑或手机上时，必须传后端机器的内网 IP
- 不需要再手改源码，只需要构建或运行时传 `--dart-define`

示例：

```bash
flutter run -d windows --dart-define=API_BASE_URL=http://192.168.1.100:3000
flutter run -d chrome --dart-define=API_BASE_URL=http://192.168.1.100:3000
flutter run -d android --dart-define=API_BASE_URL=http://192.168.1.100:3000
```

## 8. 局域网部署

### 8.1 查找后端机器内网 IP

Windows：

```powershell
ipconfig
```

Linux：

```bash
ip addr
```

假设后端机器内网地址为 `192.168.1.100`。

### 8.2 保证客户端与后端在同一网络

- 电脑客户端与后端机器要在同一局域网
- Android 手机要连接同一个 Wi-Fi
- 其他设备访问时不能再使用 `localhost`

正确示例：

```text
http://192.168.1.100:3000
```

错误示例：

```text
http://localhost:3000
```

### 8.3 放行 Windows 防火墙

如果其他设备访问不到后端，请放行 `TCP 3000` 入站规则。

建议规则：

- 类型：端口
- 协议：TCP
- 端口：`3000`
- 操作：允许连接

### 8.4 验证局域网访问

可在其他电脑浏览器或手机浏览器中访问：

```text
http://192.168.1.100:3000/
```

如果能看到 API 返回 JSON，说明网络已经打通。

## 9. Android APK 构建

### 9.1 已完成的 Android 配置

当前项目已经完成以下配置，适合内网测试：

- 在 `client/my_app/android/app/src/main/AndroidManifest.xml` 中开启了网络权限
- 允许明文 `http` 请求，便于直接访问内网后端
- 在 `client/my_app/android/app/build.gradle` 中将 `compileSdk` 提升到 `36`
- 在 `client/my_app/android/gradle.properties` 中加入了 JDK 兼容开关
- 已补充 `client/my_app/android/app/proguard-rules.pro`

### 9.2 常规构建方式

```bash
cd client/my_app
flutter pub get
flutter build apk --release --dart-define=API_BASE_URL=http://192.168.1.100:3000
```

### 9.3 JDK 17 兼容构建方式

如果本机 Flutter 或 Android Studio 自带 JDK 在构建 `release` 时出现 `jlink`、`JdkImageTransform` 或 AGP 兼容问题，可使用当前项目已经验证过的 JDK 17 构建方式：

```powershell
cd client/my_app/android
$env:JAVA_HOME='C:\path\to\jdk17'
$env:Path="C:\path\to\jdk17\bin;$env:Path"
.\gradlew.bat assembleRelease `
  -Dorg.gradle.java.home=C:\path\to\jdk17 `
  -Ptarget-platform=android-arm,android-arm64,android-x64 `
  -Pdart-defines=QVBJX0JBU0VfVVJMPWh0dHA6Ly8xOTIuMTY4LjEuMTAwOjMwMDA=
```

说明：

- `dart-defines` 的值需要按实际 `API_BASE_URL` 做 Base64 编码
- 上面的示例值对应 `http://192.168.1.100:3000`

### 9.4 APK 产物位置

```text
client/my_app/build/app/outputs/flutter-apk/app-release.apk
```

### 9.5 当前签名说明

当前 `client/my_app/android/app/build.gradle` 中的 `release` 仍然使用调试签名：

```gradle
release {
    signingConfig = signingConfigs.debug
}
```

这适合局域网测试安装，但不适合正式分发。若要正式上线，请改为正式签名或生成 AAB。

## 12. 常见问题排查

### 12.1 登录失败

优先检查：

- 输入的账号密码是否正确
- 是否使用了默认账号 `admin / admin123`
- 构建出来的前端是否指向了正确的 `API_BASE_URL`
- 手机端是否连到了后端机器所在的同一局域网
- 浏览器能否打开 `http://<服务器IP>:3000/`

### 12.2 其他电脑或手机访问不到后端

优先检查：

- 后端是否真的监听在 `0.0.0.0:3000`
- 防火墙是否放行 `TCP 3000`
- 客户端是否误用了 `localhost`
- 手机和电脑是否在同一个 Wi-Fi

### 12.3 Android Release 构建失败

优先检查：

- `JAVA_HOME` 是否指向 JDK 17
- Android SDK 是否完整
- Flutter 依赖是否已执行 `flutter pub get`
- 是否存在旧的 Gradle 缓存或 JDK 冲突

### 12.4 安装 APK 后仍连接旧后端

这通常是因为：

- 装的是旧 APK
- 重新构建时没有传 `--dart-define=API_BASE_URL=...`
- 传入的后端地址不是当前实际的内网地址

## 13. 生产环境建议

- 不要继续在仓库中保留真实数据库密码
- 将数据库连接、JWT 密钥、端口等配置迁移到环境变量
- 正式发布 Android 版本时改成正式签名
- 如果后端需要长期在线，建议使用 PM2 或系统服务托管
- 如果未来需要公网访问，建议用 Nginx 反向代理并启用 HTTPS
